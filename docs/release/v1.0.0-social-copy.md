# OpenHumming v1.0.0 Launch Copy

## GitHub release intro

OpenHumming v1.0.0 is out.

It is a local-first Python agent runtime with a visible workspace:

- `agent.md` for runtime identity
- `user.md` for durable user context
- `skills/` for reusable workflows
- `tasks/` for recurring prompts
- `summaries/` for daily review outputs

This release brings the full loop together: local UI, real tool execution, markdown memory, workflow learning, daily review, and a repeatable showcase that makes the system easy to inspect and explain.

## LinkedIn

I just shipped OpenHumming v1.0.0, a local-first Python agent runtime designed around visible evolution instead of hidden state.

The idea is simple: an agent should not only answer a prompt. It should leave behind a readable workspace that shows what it learned, what it did, and how repeated work became reusable.

OpenHumming now includes:

- CLI, HTTP, WebSocket, and local UI entrypoints
- markdown memory in `agent.md` and `user.md`
- workflow capture and skill draft promotion
- natural-language recurring tasks
- daily review summaries
- a real demo suite and evolution showcase

## Longer launch thread

1. Most agent projects still feel like prompt wrappers.
2. I wanted something that felt more like a living local system.
3. So OpenHumming stores the important parts in files you can inspect: `agent.md`, `user.md`, skills, tasks, traces, and summaries.
4. The runtime can talk, use tools, update memory, learn drafts from repeated work, and run daily review.
5. The repo now includes a real demo suite and an evolution showcase so the story is visible in minutes.
6. OpenHumming v1.0.0 is the first version where the product loop and the GitHub presentation finally line up.
