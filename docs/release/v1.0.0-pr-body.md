## Summary

OpenHumming now looks and behaves like a complete local-first agent project instead of a scaffold.
This PR closes the loop across runtime interaction, markdown memory, workflow learning, daily review, local UI, real demos, and release polish for the `v1.0.0` launch.

## What changed

- strengthened turn-level memory updates so newer collaboration preferences can replace stale entries in `agent.md` and `user.md`
- added reuse-aware draft learning so repeated successful workflows refresh existing drafts with `times_reused` evidence
- improved daily review to consolidate real activity back into memory and promote stronger draft skills
- expanded the product surface with provider settings, local UI, showcase artifacts, and the real demo suite
- upgraded the GitHub landing experience with a rewritten README, showcase docs, release copy, and visual assets

## Why

The project goal was never just "chat with tools".
It was to ship a local-first agent system with a readable workspace, a real conversation loop, visible identity and user memory, reusable skill growth, and a presentation layer strong enough to stand out on GitHub.

This PR turns that goal into a release-ready product story.

## User impact

- users can talk to the runtime through CLI, HTTP, WebSocket, or the local `/ui` page
- durable memory becomes more accurate over time instead of only accumulating bullets
- repeated successful workflows can mature into reusable skills with better promotion evidence
- the repository now includes a launch-ready demo suite and showcase for understanding the system quickly
