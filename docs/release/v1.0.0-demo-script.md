# OpenHumming v1.0.0 Demo Script

## 3-minute version

### 1. Frame the product

Say:

> OpenHumming is a local-first Python agent runtime with a visible workspace.
> Instead of hiding state in a database, it writes what it learns into `agent.md`, `user.md`, skills, traces, and summaries.

Show:

- the README hero
- the `workspace/` tree

### 2. Start the runtime

Run:

```bash
openhumming serve --host 127.0.0.1 --port 8765
```

Show:

- `http://127.0.0.1:8765/ui`

### 3. Demonstrate the live loop

Prompt:

```text
Please read `agent.md`, list the `skills` directory, then turn this workflow into skill: Workspace Orientation
```

Call out:

- the tool-assisted response
- the new draft skill
- the visible memory / skill state in the UI

### 4. Demonstrate memory updates

Prompt:

```text
From now on, please give me terse engineering updates.
```

Show:

- `user.md` gaining an interaction-style preference
- `agent.md` adapting its working style

### 5. Close with daily review and showcase

Run:

```bash
openhumming daily-review
```

Show:

- `workspace/summaries/<date>.md`
- `GET /showcase/evolution`

Close with:

> The point is not just that it answers.
> The point is that you can inspect how it changed after the answer.

## Recording shot list

1. README hero and badges
2. Workspace tree
3. Local UI loading
4. Workflow-to-skill draft interaction
5. Memory preference update
6. Daily review summary
7. Evolution showcase cards

## Backup plan

If live APIs are unavailable, switch to the local provider and run:

```bash
python scripts/run_real_demos.py
```

This still demonstrates the full product loop with deterministic local outputs.
