# OpenHumming v1.0.0

OpenHumming is now a complete local-first agent loop rather than a scaffold.
This release turns the project into a more legible, demoable, and publishable system: it can talk through a local port, persist memory in markdown, learn reusable skills from repeated workflows, schedule future prompts, and consolidate the day through daily review.

## Highlights

- Local chat loop through CLI, HTTP, WebSocket, and `/ui`
- Markdown memory in `agent.md` and `user.md`
- Reuse-aware skill learning with reviewable drafts and promotion
- Natural-language recurring tasks with APScheduler-backed execution
- Real demo suite plus before/after evolution showcase
- Release-ready docs, launch copy, and GitHub presentation assets

## What Changed

### Runtime and memory

- Added stronger turn-level memory updates that can replace stale collaboration preferences instead of only appending new bullets.
- Improved daily review so it consolidates real conversation evidence back into durable agent and user memory.
- Exposed richer memory proposal and applied-update payloads through the API and WebSocket surfaces.

### Skill learning

- Added workflow capture and draft evaluation paths for tool-assisted work.
- Updated learned drafts when the same workflow succeeds again, including `times_reused` evidence.
- Enabled daily review to promote stronger drafts into published skills.
- Allowed nested folders under `workspace/skills/` to act as extension packs.

### Product surface

- Added a lightweight local `/ui` console for live interaction.
- Added provider switching from the browser without editing `.env`.
- Added an evolution showcase endpoint and in-repo showcase docs.
- Added a repeatable real demo suite under `examples/real_demos/`.

### Release polish

- Rewrote the README as a launch-focused project homepage.
- Added SVG presentation assets for the GitHub landing page and showcase story.
- Added a release kit with PR copy, release notes, demo script, and launch posts.

## Why It Matters

OpenHumming is designed to feel like a living local system instead of a prompt wrapper:

- you can inspect what it knows
- you can diff how it changes
- you can watch successful work turn into reusable skill assets
- you can run daily review and see the loop close
